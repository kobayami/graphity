# Benchmark Baselines

This directory holds the **published** benchmark reports — one per minor release. They are the data source the project links to from `README.md` and the source of truth for all externally-cited performance figures.

## Convention

- One file per minor version: `1.0.md`, `1.1.md`, `1.2.md`, …
- Generated by `../run-benchmarks.sh all full` on the corresponding release tag's commit (`v1.0.0`, `v1.1.0`, …) and copied here verbatim. The header block in each file (timestamp, JDK, OS, CPU, RAM) records the exact run conditions.
- Patch releases (`1.0.1`, `1.0.2`, …) reuse their minor's baseline unless an explicit re-measurement is warranted; in that case a `1.0.1.md` may be added alongside `1.0.md`.

## Difference from `../results/`

`../results/` holds **local, throw-away** benchmark output (gitignored). It is what the runner script writes during day-to-day development. Files there are unrelated to released figures.

`baselines/` holds **curated, version-pinned** results that ship with the repo and are linked from end-user documentation. New baselines are added here only when a new minor release is cut.

## Reproducing a baseline

```bash
git checkout v1.0.0
mvn -am -pl graphity-benchmarks package -DskipTests
./graphity-benchmarks/run-benchmarks.sh all full
# Output lands in graphity-benchmarks/results/, then is copied here for the release.
```

Hardware variability is unavoidable; the relative factors (Graphity vs. JGraphT/Guava/WebGraph) are what should reproduce, not the absolute millisecond figures.
